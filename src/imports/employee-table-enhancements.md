Завдання — привести header відділів та рядок Resources до більш компактного і зрозумілого вигляду, не змінюючи layout таблиці співробітників.

1. Спростити header відділу.

З header відділу прибрати індикатор нестачі годин (−XXг), щоб уникнути дублювання інформації.

Залишити:

назву відділу  
кількість працівників  

Приклад:

Ресепшн   3

Індикатори відкритих змін та інших статусів справа залишити без змін.


2. Оновити ліву частину рядка Resources.

Замість трьох текстових блоків:

Потрібно XXг  
Заплановано XXг  
Нестача XXг  

показувати компактний формат:

XXг   XXг   ±XXг

де:

перше число — потрібно (forecast)  
друге число — заплановано (scheduled)  
третє число — різниця  

Приклад:

144г   102г   −42г

При наведенні показувати tooltip з підписами:

Потрібно: XXг  
Заплановано: XXг  
Різниця: ±XXг


3. Залишити індикатори покриття по днях у рядку Resources.

У колонках днів (Пн–Нд) показувати співвідношення:

scheduled / forecast

Приклад:

22/24
16/24
24/24
8/16

Кольорову заливку клітинок залишити:

зелений — покриття достатнє  
жовтий — близько до нестачі  
червоний — нестача

Tooltip при наведенні:

Потрібно: XXг  
Заплановано: XXг  
Різниця: ±XXг


4. Не змінювати рядок "Відкриті зміни".

Залишити текст "Відкриті зміни" та поточну поведінку.


5. Не змінювати структуру grid зі співробітниками.

Карточки змін, рядки співробітників і layout таблиці повинні залишитися без змін.


Результат:

header відділу стає компактнішим  
рядок Resources зберігає детальну інформацію  
у колонках днів видно покриття змін по кожному дню  
grid співробітників залишається без змін.
