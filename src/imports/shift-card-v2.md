# User Story: Оптимізована картка зміни (Shift Card v2)

## Епік

Weekly Schedule UI / Shift Visualization

---

# 📖 Опис

Як менеджер зміни
Я хочу бачити компактні та легко читабельні картки змін у таблиці розкладу
Щоб швидко сканувати графік працівників, не читаючи великі текстові блоки.

Картка повинна показувати **ключову інформацію миттєво**, а деталі — у tooltip.

---

# 🎯 Основна проблема поточної реалізації

Поточна картка перевантажена текстом:

* показує деталі підрозділів
* дублює інформацію
* складно швидко сканувати

Через це менеджеру потрібно **читати кожну картку**, а не **розпізнавати її візуально**.

---

# 🧩 Нова анатомія картки

Картка повинна містити **3 інформаційні рівні**.

## 1. Основна інформація

Перший рядок:

```
08:00–16:00
```

Стиль:

* text-sm
* font-semibold
* foreground

Праворуч можуть з’являтись **індикатори стану**.

---

## 2. Підрозділ

Другий рядок:

```
Ресепшн
```

Якщо кілька підрозділів:

```
Ресепшн +1
```

Повна деталізація показується у tooltip.

Стиль:

* text-xs
* muted-foreground

---

## 3. Візуальний індикатор зміни

Під текстом відображається **Shift Timeline Bar**.

Це горизонтальна смуга, яка показує:

* робочі сегменти
* перерву
* різні підрозділи

Приклад:

```
████░░██████
```

Правила:

* кожен сегмент відповідає частині зміни
* колір сегмента = підрозділ
* перерва = нейтральний сегмент

---

# 🎨 Поведінка Timeline Bar

Timeline бар будується з subUnits та break.

Наприклад:

```
08:00–12:00 Ресепшн A
12:00–12:30 перерва
12:30–16:00 Ресепшн B
```

Відобразиться як:

```
████░░████
```

де:

* синій = Ресепшн A
* grey = break
* інший синій = Ресепшн B

---

# ⚡ Індикатори стану

У правій частині першого рядка можуть з’являтись маленькі іконки.

| Іконка | Значення          |
| ------ | ----------------- |
| ⚡      | зміна з біржі     |
| ✏      | зміна редагується |

* | відкрита зміна |
  ! | конфлікт |

---

# 🏝 Відсутності

Відсутності повинні мати окремий вигляд.

## Відпустка

```
🏖 Відпустка
Весь день
```

## Лікарняний

```
🤒 Лікарняний
Весь день
```

Ці картки:

* не draggable
* не мають timeline bar
* мають accent background

---

# 🧠 Деталізація у tooltip

Hover по картці показує повну деталізацію:

```
08:00–16:00

08:00–12:00 — Ресепшн A
12:00–12:30 — Перерва
12:30–16:00 — Ресепшн B
```

Tooltip також показує:

* тривалість сегментів
* підрозділи

---

# 🎨 Візуальні правила

Картка повинна бути **легшою та компактнішою**.

Зміни:

* прибрати текстові розшифровки сегментів
* залишити тільки назву підрозділу
* деталізацію перенести у tooltip

Висота картки повинна зменшитися.

---

# ⚙️ Інтерактивність

Зберегти існуючу поведінку:

Click → відкриває PlanningDrawer
Right-click → контекстне меню
Drag → drag & drop

---

# ✅ Acceptance Criteria

Картка містить:

* timeRange
* назву підрозділу
* timeline bar

Картка не містить:

* детальної текстової розшифровки сегментів

Timeline bar:

* показує break
* показує кілька підрозділів

Tooltip:

* показує повну структуру зміни

Absence cards:

* мають окремий вигляд
* не draggable

Drag & drop:

* працює як у поточній реалізації
